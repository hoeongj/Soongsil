# 🗄️ File Processing (파일 처리)

저수준 파일 입출력 시스템 콜부터 **플래시 메모리(FTL)** 시뮬레이션, **가변길이 레코드 기반 파일 DB**까지, 파일을 다루는 다양한 계층을 C로 구현한 과목입니다.

> **기술 스택:** C · POSIX 파일 I/O(`open`/`read`/`write`/`lseek`) · `fread`/`fwrite` 바이너리 I/O · 비트/바이트 단위 데이터 처리

---

## 📂 01. 저수준 파일 I/O (`01_LowLevel_File_IO/`)
파일 디스크립터 기반 시스템 콜(`open`/`read`/`write`/`lseek`)만으로 파일을 조작하는 유틸리티 모음입니다.

| 파일 | 기능 |
|------|------|
| `copy.c` | 파일 복사 (버퍼 단위 read/write) |
| `insert.c` | 지정 위치에 데이터 삽입 |
| `delete.c` | 지정 구간 바이트 삭제 |
| `overwrite.c` | 지정 위치 덮어쓰기 |
| `read.c` | 지정 범위 읽기 |
| `merge.c` | 두 파일 병합 |
| `rprint.c` | 파일 내용 역순 출력 |

- **핵심:** `lseek`로 파일 오프셋을 직접 제어하고, 표준 입출력(stdio) 버퍼링 없이 **시스템 콜 수준에서 바이트를 조작**.

## 📂 02. FTL — Flash Translation Layer (`02_Flash_Translation_Layer/`)
플래시 메모리의 물리적 특성을 반영한 **FTL 일부 기능**을 구현했습니다. (`ftl.c`)

- **과제 목표:** 플래시 메모리 에뮬레이터에 대해 **페이지 단위 읽기/쓰기, 블록 단위 소거**를 수행하고, 강의자료의 **in-place update** 방식으로 갱신을 구현한다.
- **구현 내용:**
  - `c`(생성): 블록 수만큼 `erase`로 `0xFF` 초기화하여 플래시 파일 생성
  - `w`(쓰기): 페이지 버퍼를 `0xFF`로 초기화 후 섹터 데이터 + 스페어(정수, 바이너리) 기록 → `fdd_write`
  - `r`(읽기): 페이지를 읽어 섹터 데이터와 스페어 데이터를 **분리**하여 출력
  - `e`(소거): 블록 단위 `fdd_erase`
  - **`u`(in-place update):** 플래시는 덮어쓰기가 불가능하므로 → ① 대상 블록의 유효 페이지를 **예약 블록으로 복사** → ② 원본 블록 소거 → ③ 예약 블록에서 다시 복원 → ④ 빈 자리에 새 데이터 기록 → ⑤ 예약 블록 소거. 이 과정의 **read/write/erase 횟수를 카운트**하여 출력.
- **핵심:** 플래시 메모리의 *erase-before-write* 제약, 페이지/블록 단위 연산, 스페어 영역, 가비지 컬렉션과 유사한 블록 재배치 개념을 코드로 이해.

## 📂 03. 가변길이 레코드 파일 DB (`03_VariableLength_Records/`)
파일 하나를 데이터베이스처럼 다루어 **가변길이 학생 레코드**를 관리합니다. (`student.c`, `student.h`)

- **레코드 구조:** `길이 지시자(2) + 학번 + 이름 + 학과 + 주소 + 이메일`을 `#` 구분자로 직렬화(`id#name#dept#addr#email#`).
- **파일 헤더:** `레코드 수(4B) + 리스트 헤드(4B)` — 삭제된 레코드를 잇는 **free list** 관리에 사용.
- **구현 내용:** `readHeader`/`writeHeader`로 헤더를 관리하고, `makeRecordData`로 레코드를 직렬화(길이 검증 포함). 가변길이 레코드의 삽입·삭제·조회를 오프셋 기반으로 처리.
- **핵심:** 고정길이가 아닌 **가변길이 레코드의 직렬화/역직렬화**, 길이 지시자, free list를 통한 공간 재사용 등 파일 기반 저장 구조 설계.

---

## 핵심 기술
- POSIX 시스템 콜(`open`/`read`/`write`/`lseek`)과 바이너리 파일 I/O
- 플래시 메모리 특성(erase-before-write, 페이지/블록)과 **FTL in-place update**
- 가변길이 레코드 직렬화, 파일 헤더·free list 설계
- 바이트/오프셋 단위의 정밀한 데이터 조작

> 📝 FTL은 강의에서 제공한 프레임워크(`flash.h`, `fdevdriver.c`)의 디바이스 드라이버 인터페이스(`fdd_read`/`fdd_write`/`fdd_erase`)를 사용하며, 본 저장소에는 직접 작성한 `ftl.c`가 포함되어 있습니다.
