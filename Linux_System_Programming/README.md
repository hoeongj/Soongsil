# 🐧 Linux System Programming (리눅스 시스템 프로그래밍)

POSIX 시스템 콜을 직접 다루어 **자동 채점 시스템**과 **MD5 기반 중복 파일 탐색·정리 도구**를 C로 구현한 과목입니다. 프로세스 생성·제어, 시그널, I/O 리다이렉션, 디렉토리 트리 순회, 해시, 동적 자료구조 등 시스템 프로그래밍의 핵심을 모두 활용했습니다.

> **기술 스택:** C · POSIX(`fork`/`exec`/`wait`, `dup2`, `alarm`/signal, `dirent`, `stat`) · pthread · OpenSSL(MD5 EVP) · Makefile

## 📂 Project 1 — `ssu_score` : 자동 채점 시스템

`project1_ssu_score/` · 소스: [`source/`](./project1_ssu_score/source) · 보고서: `report.docx` · 명세: `spec.pdf`

### 과제 목표
학생들이 제출한 답안 디렉토리를 **정답 디렉토리와 자동으로 비교·채점**하고, 학생별 점수표를 산출하는 채점 프로그램을 작성한다. 두 가지 문제 유형을 모두 지원해야 한다.
1. **빈칸 채우기(`.txt`)** 문제 — 텍스트 정답과 비교
2. **프로그램(`.c`)** 문제 — 학생 코드를 **컴파일·실행**하여 출력 결과를 정답과 비교

### 구현 내용
- **`main.c`** — `gettimeofday()`로 전체 채점 실행 시간을 측정하고, `ssu_score()`를 호출한 뒤 소요 시간을 출력한다.
- **`ssu_score.c`** — 채점 엔진. 학생 답안 디렉토리(`stuDir`)와 정답 디렉토리(`ansDir`)를 `opendir`/`readdir`로 순회하며 문제별로 채점하고, `score_table`·`id_table`에 학생별 점수를 누적한다.
- **`blank.c` / `blank.h`** — 빈칸 채우기(`.txt`) 문제 채점 모듈.
- 프로그램 문제는 **`fork()` + `exec()`** 로 자식 프로세스에서 학생 프로그램을 실행하고, **`dup2()`** 로 표준 출력/에러를 파일로 리다이렉션하여 결과를 캡처한 뒤 정답과 비교한다.
- 무한 루프·시간 초과 제출물에 대비해 **`alarm()`/시그널 기반 타임아웃(5초)** 으로 자식 프로세스를 강제 종료하여 채점이 멈추지 않도록 했다.
- 다수 학생 답안을 동시에 처리하기 위해 **스레드**(`threadFiles`, `cIDs`)를 활용했다.

### 핵심 기술
프로세스 생성·제어(`fork`/`exec`/`waitpid`), I/O 리다이렉션(`dup2`), 시그널·타임아웃(`alarm`), 디렉토리 순회(`opendir`/`readdir`), 파일 입출력, 멀티스레딩, 실행 시간 측정.

### 빌드 & 실행
```bash
cd project1_ssu_score/source
make
./ssu_score <학생답안_디렉토리> <정답_디렉토리>
```

## 📂 Project 2 — `ssu_find-md5` : 중복 파일 탐색·정리 도구 (≈2,300줄)

`project2_ssu_find-md5/` · 소스: [`ssu_find-md5.c`](./project2_ssu_find-md5/ssu_find-md5.c), `ssu_clean.c`, `ssu_help.c` · 보고서: `report.docx` · 명세: `spec.pdf`

`fdupes`와 유사한 **대화형 파일 관리 셸**입니다. 프롬프트에서 명령을 입력하면 디렉토리 트리에서 내용이 동일한 파일을 찾아 묶고, 옵션에 따라 정리(삭제)합니다.

### 과제 목표
지정한 경로 이하를 탐색하여 **내용이 완전히 동일한 파일(중복 파일)** 들을 찾아 "중복 세트"로 묶고, 다양한 필터·정렬·삭제 옵션을 제공하는 명령어를 구현한다.

### 구현 내용 — `fmd5` 명령 (`ssu_find-md5.c`)
- **2단계 중복 판별:** 먼저 **파일 크기**로 1차 그룹핑하고, 같은 크기 그룹 안에서 **MD5 해시**가 같은 파일만 최종 중복 세트로 확정하여 불필요한 해시 계산을 줄였다.
- **BFS 디렉토리 순회:** 디렉토리 큐(`DirQueue`)를 두고 너비 우선으로 하위 디렉토리를 탐색하며, `-d` 깊이 제한과 `/proc`·`/run`·`/sys` 같은 가상 파일시스템 자동 제외를 구현.
- **MD5 해시:** **OpenSSL EVP** API(`EVP_DigestInit/Update/Final`)로 파일을 8KB 청크 단위로 읽어 해시를 누적 → 대용량 파일도 메모리 부담 없이 처리. (헤더가 없는 로컬 환경을 위한 최소 EVP 선언 폴백까지 작성)
- **풍부한 필터 옵션을 직접 파싱:**
  - 확장자(`*`, `*.txt`), 크기 범위(`B`/`KB`/`MB`/`GB`, 무제한 `~`, 소수점 검증),
  - 접근/수정/변경 시각 범위(`YYYY:MM:DD:HH` 파싱, **윤년·월별 일수 보정**, from/to 경계 처리),
  - 권한(8진수 `mode`), 하드링크 제외, 제외 디렉토리(`-e`, 최대 3개).
- **동적 자료구조:** 파일 노드(`FileNode`) → 중복 세트(`FileSet`) → 세트 목록(`SetList`)의 3계층 연결 리스트를 직접 구현하고, **배열 없이 연결 리스트만으로 삽입 정렬**(경로 길이·사전순)을 수행. 모든 `malloc`에 대응하는 `free`로 메모리 누수 방지.
- **정렬·삭제:** 크기/수정시각/경로 기준 정렬(`-r`), 중복 개수 하한(`-x`), 출력 개수 제한(`-n`), 그리고 삭제 시 **보존 규칙**(최신·최오래·최단경로·최장경로)에 따라 한 파일만 남기고 정리.

### 구현 내용 — `ssu_clean`(메인/정리), `ssu_help`(도움말)
- `ssu_clean.c` 가 대화형 프롬프트와 명령 분기를 담당하고, `ssu_help.c` 가 사용법을 출력한다.

### 핵심 기술
BFS 트리 순회, MD5(OpenSSL EVP), 다단계 연결 리스트 설계와 삽입 정렬, 견고한 문자열/숫자/날짜 파싱(방어적 입력 검증), `stat`·`realpath`·홈 디렉토리 확장, 동적 메모리 관리.

### 빌드 & 실행
```bash
cd project2_ssu_find-md5
make
./ssu_clean          # 대화형 프롬프트 진입 후 fmd5/help 등 명령 입력
```

> ⚙️ MD5 계산에 `libcrypto`(OpenSSL)가 필요합니다: `gcc ... -lcrypto`
