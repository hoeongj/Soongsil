# 🐧 Linux System Programming (리눅스 시스템 프로그래밍)

POSIX 시스템 콜을 직접 다루어 **자동 채점 시스템**과 **MD5 기반 중복 파일 탐색·정리 도구**를 C로 구현한 과목입니다. 프로세스 생성·제어, 스레드 병렬화, 시그널·타임아웃, I/O 리다이렉션, 디렉토리 트리 순회, 해시, 동적 자료구조 등 시스템 프로그래밍의 핵심을 모두 활용했습니다.

> **기술 스택:** C · POSIX(`fork`/`execvp`/`waitpid`, `dup2`, `alarm`/signal, `dirent`, `stat`) · **pthread** · OpenSSL(MD5 EVP) · Makefile

## 📂 Project 1 — `ssu_score` : 자동 채점 시스템 (⭐ 성능 최적화 프로젝트)

`project1_ssu_score/` · 소스: [`source/`](./project1_ssu_score/source) · 보고서: `report.docx` · 명세: `spec.pdf`

### 과제 목표
학생 답안 디렉토리를 **정답 디렉토리와 자동 비교·채점**하여 `score.csv`로 점수표를 산출하는 채점기를 작성한다. 두 가지 문제 유형을 모두 지원한다.
1. **빈칸 채우기(`.txt`)** — 텍스트 정답과 비교(괄호 검사·토큰화·구문 트리 비교까지 수행)
2. **프로그램(`.c`)** — 학생 코드를 **컴파일·실행**하여 표준출력 결과를 정답 실행 결과와 비교

> 💡 **이 프로젝트의 핵심은 "정확성"이 아니라 "수행 속도"였습니다.** 채점 결과가 정답과 일치(PASS)하는 것을 전제로, **전체 채점에 걸리는 실행 시간**을 측정해 그 빠르기로 점수를 매겼습니다(`main.c`의 `gettimeofday` 기반 런타임 측정). 따라서 **느리게 동작하는 기준(baseline) 구현을 얼마나 빠르게 최적화하느냐**가 과제의 본질이었습니다.

### 🚀 성능 최적화 — 순차 채점(baseline) → 병렬 파이프라인
동작하지만 느린 **순차 채점 기준 구현**을 출발점으로, 다음 6가지를 적용해 대규모로 재설계했습니다. (모든 변경 지점은 소스에 `[개선]` 주석으로 명시)

| # | 최적화 | 무엇을 / 왜 | 효과 |
|---|--------|-------------|------|
| 1 | **순차 → 병렬 채점 (pthread)** | 학생을 1명씩 채점하던 것을 **학생마다 스레드**(`thread_grade`)로 동시 채점 | 멀티코어 활용, 채점 시간 대폭 단축 |
| 2 | **사전 컴파일·실행 파이프라인** | 채점 시점마다 `gcc` 호출하던 것을, 모든 학생×문제를 **미리 `fork`로 병렬 컴파일+실행**(`precompile_students`)하고 결과(`.stdout`)만 저장 | 채점 단계에서 `gcc` 호출 제거, 비교만 수행 |
| 3 | **파이프라인 오버랩** | `score_students`에서 `fork`로 **정답 전처리(`preprocess_answers`)와 학생 전처리를 동시에** 수행 | 두 준비 작업의 시간이 겹쳐 단축 |
| 4 | **정답 캐싱 (반복 I/O 제거)** | 빈칸 정답을 매 학생마다 다시 읽던 것을 **메모리에 1회 캐싱**(`load_blank_cache`) | 학생 수만큼의 파일 open/read/close 제거 |
| 5 | **`system()` → `fork`/`execvp`** | 비용 크고 스레드-비안전한 `system()` 대신 직접 프로세스 제어 | **스레드 안전성** 확보 + 오버헤드 감소 |
| 6 | **블록 단위 I/O + 출력 버퍼링** | 1바이트씩 `read`하던 `get_answer`를 **블록 단위 read + `lseek`** 로, 매 컬럼 `write`를 **버퍼 축적 후 단일 `write`** (`thread_grade`/`write_first_row`) | 시스템 콜 횟수 급감 |

#### 4단계 병렬 파이프라인 (`score_students`)
```
1) score.csv 생성 + 헤더(단일 write)
2) [fork] 자식: 정답 컴파일·실행   ┐ 동시 진행
        부모: 빈칸 캐시 + 학생 프로그램 사전 컴파일·실행 ┘ → waitpid 동기화
3) 학생마다 pthread_create → thread_grade 로 병렬 채점
4) pthread_join 후, 스레드별 버퍼(csv_buf)를 순서대로 기록
```

#### 견고성(robustness)도 함께 설계
- **타임아웃(`wait_batch`):** 학생 프로그램이 무한 루프/입력 대기로 멈추지 않도록, `waitpid(WNOHANG)` 비차단 폴링으로 대기하다 **5초 초과 시 `SIGKILL`** 로 강제 종료(좀비 회수 포함). `usleep(5ms)`로 폴링 CPU 과부하 방지.
- 자식 표준입력을 `/dev/null`로 리다이렉트해 입력 대기로 인한 영구 블록 차단.
- 원본의 버그(`fclose` 누락, 타임아웃 처리 등)도 함께 수정.

### 🏆 측정 결과
- **정확성:** 채점 결과가 정답과 일치 → **PASS** (3회 측정 모두)
- **수행 시간:** **약 5.73초** (측정값 5.742s / 5.732s / 5.732s)
- **성적:** 수행 시간 구간별 배점에서 **가장 빠른 구간(5초대)** 에 해당하여 **수행 시간 점수 만점(70/70)** 획득.

> 참고 — 채점은 *(빠를수록 고득점)* 구간 방식이었습니다: **5초대 → 70점(만점)**, 10초대 → 65점, 15초대 → 60점, 20초대 → 55점, … 180초 이내 → 25점. 위 최적화로 최상위 구간을 달성했습니다.

### 핵심 기술
프로세스 생성·제어(`fork`/`execvp`/`waitpid`/`WNOHANG`), **POSIX 스레드 병렬화(pthread)**, I/O 리다이렉션(`dup2`), 시그널·타임아웃(`SIGKILL`/`alarm`), 디렉토리 순회(`opendir`/`readdir`), 블록 단위 파일 I/O, 메모리 캐싱·버퍼링, 실행 시간 측정·프로파일링.

### 빌드 & 실행
```bash
cd project1_ssu_score/source
make
./ssu_score <학생답안_디렉토리> <정답_디렉토리>     # 채점 후 score.csv 생성
# 옵션: -p(점수·평균 출력) -e <dir>(에러파일 분리) -t <문제>(-lpthread 컴파일) -c <학번>(점수 조회) -h(도움말)
```

## 📂 Project 2 — `ssu_find-md5` : 중복 파일 탐색·정리 도구 (≈2,300줄)

`project2_ssu_find-md5/` · 소스: [`ssu_find-md5.c`](./project2_ssu_find-md5/ssu_find-md5.c), `ssu_clean.c`, `ssu_help.c` · 보고서: `report.docx` · 명세: `spec.pdf`

`fdupes`와 유사한 **대화형 파일 관리 셸**입니다. 프롬프트에서 명령을 입력하면 디렉토리 트리에서 내용이 동일한 파일을 찾아 묶고, 옵션에 따라 정리(삭제)합니다.

### 과제 목표
지정 경로 이하를 탐색하여 **내용이 완전히 동일한 파일(중복 파일)** 을 "중복 세트"로 묶고, 다양한 필터·정렬·삭제 옵션을 제공하는 명령(`fmd5`)을 구현한다.

### 구현 내용 — `fmd5` 명령 (`ssu_find-md5.c`)
- **2단계 중복 판별:** 먼저 **파일 크기**로 1차 그룹핑하고, 같은 크기 그룹 안에서만 **MD5 해시**를 계산·비교해 최종 중복 세트를 확정 → 불필요한 해시 연산 최소화.
- **BFS 디렉토리 순회:** 디렉토리 큐(`DirQueue`)로 너비 우선 탐색, `-d` 깊이 제한, `/proc`·`/run`·`/sys` 가상 파일시스템 자동 제외.
- **MD5 해시:** **OpenSSL EVP**(`EVP_DigestInit/Update/Final`)로 파일을 **8KB 청크 단위**로 읽어 누적 → 대용량 파일도 메모리 부담 없이 처리. (헤더 없는 환경용 최소 EVP 선언 폴백까지 작성)
- **풍부한 필터를 직접 파싱·검증:** 확장자(`*`, `*.txt`), 크기 범위(`B`/`KB`/`MB`/`GB`, 무제한 `~`, 소수점 검증), 접근/수정/변경 시각(`YYYY:MM:DD:HH`, **윤년·월별 일수·from/to 경계 보정**), 권한(8진수), 하드링크 제외, 제외 디렉토리(`-e`, 최대 3개).
- **3계층 동적 자료구조:** 파일 노드(`FileNode`) → 중복 세트(`FileSet`) → 세트 목록(`SetList`)을 모두 연결 리스트로 구현하고, **배열 없이 연결 리스트만으로 삽입 정렬**(경로 길이·사전순). 모든 `malloc`에 대응하는 `free`로 누수 방지.
- **정렬·삭제:** 크기/수정시각/경로 기준 정렬(`-r`), 중복 개수 하한(`-x`), 출력 개수 제한(`-n`), 삭제 시 **보존 규칙**(최신·최오래·최단/최장 경로)으로 한 파일만 남기고 정리.

### 구현 내용 — `ssu_clean`(메인/정리), `ssu_help`(도움말)
`ssu_clean.c`가 대화형 프롬프트와 명령 분기를, `ssu_help.c`가 사용법 출력을 담당합니다.

### 핵심 기술
BFS 트리 순회, MD5(OpenSSL EVP), 다계층 연결 리스트 설계와 삽입 정렬, 방어적 문자열·숫자·날짜 파싱, `stat`·`realpath`·홈 디렉토리 확장, 동적 메모리 관리.

### 빌드 & 실행
```bash
cd project2_ssu_find-md5
make
./ssu_clean            # 대화형 프롬프트 진입 후 fmd5/help 등 입력
```
> ⚙️ MD5 계산에 OpenSSL이 필요합니다: `gcc ... -lcrypto`
